Two paths, one topic — Avro + Apicurio vs. raw JSON

A dummy end-to-end walkthrough of a producer publishing an event and a consumer reading it back — once through Avro serialization validated against an Apicurio schema registry, and once as a plain JSON string with no contract at all. Press play to watch a payload travel each pipeline stage by stage.

Read it as: Lane A registers/looks up the Avro schema in Apicurio, gets a compatibility check (BACKWARD by default), then ships a compact binary payload tagged with a schema ID. The consumer resolves that ID back to the exact schema before deserializing. Lane B just stringifies an object and ships it — nothing checks that the consumer's expectations still match what the producer sent.
This is an illustrative simulation for teaching the concept, not a live Kafka cluster.
